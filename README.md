# Приложение для создания маркеров на карте с поддержкой загрузки изображений (Lab2)

### Требования

Nojde.js (версия v24.13 и новее)

Expo Go for SDK version 54

### Установка и запуск приложения

Клонирование репозитория: "git clone https://github.com/NPNN00/mobile_dz2.git"

Переход в папку проекта: "cd mobile_dz2"

Загрузка и обновление зависимостей: "npm install expo"

Запуск приложения: "npx expo start"

### Запуск на смартфоне

Сканировать QR-код через Expo Go или через стандартный сканер на телефоне

## Принятые решения при реализации

#### Файловая маршрутизация через expo-router:

**Два экрана:**

- **app/index.tsx** для просмотра карты с маркерами
- **app/marker/[id].tsx** для просмотра и редактиования маркера

**Локальное хранилище:**

Для сохранения данных маркеров используется SQLite (expo-sqlite).

База данных markers.db содержит две таблицы:

- markers: хранит информацию о маркерах (id,latitude,longitude,title,description,created_at)
- marker_images: хранит изображения маркеров (id,marker_id,uri,created_at) с внешним ключом и каскадным удалением

**Экраны и компоненты:**

- app/index.tsx: отображает компоненту Map.tsx
  - Map.tsx: отображение карты (\<MapView>). Отображает компоненту MarkerList.tsx. Использует хук useFocusEffect для обновления списка маркеров.
    - MarkerList.tsx: загрузка маркеров из AsyncStorage и их отображение (\<Marker>).

- app/marker/[id].tsx: просмотр или редактирование маркера. Использует Хук useLocalSearchParams для получения id и координаты метки. Использует хук useEffect, зависимый от состояния редактирования метки (isEditing), для загрузки актуальных данных метки. Использует хук useSafeAreaInsets для отступа контента от шторки и навигационных кнопок телефона
  - ImageList.tsx: компонента для отображения изображений маркера, который работает в режиме просмотра и редактирования

**Контекст и работа с БД:**

- DatabaseContext.tsx: предоставляет контекст для работы с базой данных во всём приложении
- schema.ts: инициализация БД и создание таблиц
- operations.ts: содержит все необходимые операции с БД

Структура DatabaseContext.tsx:

- DatabaseProvider: компонент-провайдер, оборачивающий приложение и ицилизирующий БД
- useDatabase - кастомный хук для доступа к контексту с проверкой на использование вне провайдера

Управление состоянием:

- db - экземпляр подключения к SQLite
- isLoading - флаг, блокирующий интерфейс во время операции записи/обновления/удаления

Инициализация: при монтировании провайдера создаются таблицы (если их нет) и устанавливается подключение к БД

Каррирование операций: методы для операций над БД создаются в два этапа - сначала принимают зависимости (db,setIsLoading), затем параметры операции. Это позволяет вынести методы вне DatabaseContext

**Обработка ошибок:**

- При создании и обновлении маркера используется единые транзакции для маркера и всех связанных изображений (addMarkerWithImages и updateMarkerWithImages)
